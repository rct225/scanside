Device Properties

Device properties are stored in the system and read by the driver when it loads, and then the device is programmed according to these loaded values. The properties are protected during read and write access times, and some of them are pre-defined by the configuration dialog box. Different device's functions are influenced by different properties. These properties are represented by an ID code for an application that calls Advantech API functions. An application can use a default value set by the configuration dialog box, or call Advantech API functions: DRV_DeviceGetProperty and DRV_DeviceSetProperty, to modify individual properties in running time.

 

PCM-3761I supports the following properties:

Property Description
CFG_BoardID gets the board ID of the device.
CFG_BusNumber gets bus number of the currently installed PCM-3761I.
CFG_SlotNumber gets the slot number of the currently installed PCM-3761I.
CFG_Interrupt gets the IRQ of the currently installed PCM-3761I.
CFG_BaseAddress gets the base address of the currently installed PCM-3761I.
CFG_DiChannelCount gets the total count of DI channels the currently installed PCM-3761I.
CFG_DoChannelCount gets the total count of DO channels of the currently installed PCM-3761I.
CFG_DiPortCount gets the total count of DI ports of the currently installed PCM-3761I.
CFG_DoPortCount gets the total count of DO ports of the currently installed PCM-3761I.
CFG_DiInterruptTriggerOnRisingEdge gets or sets the DI channel which issue interrupt on rising trigger mode of the currently installed PCM-3761I.
CFG_DiInterruptTriggerOnFallingEdge gets or gets the DI channel which issue interrupt on falling edge trigger mode of the currently installed PCM-3761I.
