Device Properties

Device properties are stored in the system and read by the driver when it loads, and then the device is programmed according to these loaded values. The properties are protected during read and write access times, and some of them are pre-defined by the configuration dialog box. Different device's functions are influenced by different properties. These properties are represented by an ID code for an application that calls Advantech API functions. An application can use a default value set by the configuration dialog box, or call Advantech API functions: DRV_DeviceGetProperty and DRV_DeviceSetProperty, to modify individual properties in running time.

PCI-1757UP supports the following properties:

 

Property Description
CFG_BoardID gets the board ID of the device.
CFG_BaseAddress gets the base address of the currently installed PCI-1757UP.
CFG_Interrupt gets the IRQ of the currently installed PCI-1757UP.
CFG_BusNumber gets bus number of the currently installed PCI-1757UP.
CFG_SlotNumber gets the slot number of the currently installed PCI-1757UP.
CFG_DioChannelDirection gets/sets the DI/O ports' direction of the currently installed PCI-1757UP.
CFG_DiChannelCount gets the digital input channel count of the currently installed PCI-1757UP.
CFG_DoChannelCount gets the digital output channel count of the currently installed PCI-1757UP.
CFG_DiPortCount gets the digital input port count of the currently installed PCI-1757UP.
CFG_DoPortCount gets the digital output port count of the currently installed PCI-1757UP.
CFG_InterruptTriggerSource gets/sets interrupt trigger source of the currently installed PCI-1757UP.
CFG_DiInterruptTriggerOnRisingEdge One bit for one channel. If bit value is 1, the channel will issue interrupt on rising edge, otherwise issue interrupt on falling edge..
CFG_DiInterruptTriggerOnFallingEdge One bit for one channel. If bit value is 1, the channel will issue interrupt on falling edge, otherwise issue interrupt on rising edge.
